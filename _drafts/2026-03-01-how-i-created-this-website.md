# How I Created This Website

This sketch is based on chat-history intent: clean up old articles and publish many more.

## 1. The real problem I was solving

I did not start with "how do I build a website?"
I started with a different question:
How do I keep writing for years without losing momentum?

I needed a system where:

- drafts are easy to start
- publishing is simple
- old writing can be cleaned and upgraded over time

## 2. Core idea: writing system, not just a website

The site is a container for long-term thinking.
It is less about web design trends and more about consistency.
I wanted a quiet reading experience and a workflow I can sustain.

## 3. Build sketch (high level)

1. Pick a static, low-friction stack.
2. Shape the layout for long-form reading.
3. Keep drafting and publishing as markdown-first.
4. Use a drafts pipeline instead of perfect-first publishing.
5. Improve quality iteratively through cleanup passes.

## 4. Why the drafts folder matters

The key decision was to treat drafts as a permanent workbench, not a temporary folder.
This allows:

- rough ideas to exist without pressure
- partial articles to mature over time
- batch cleanup when I am in editing mode

## 5. The current phase

Right now, I am entering a new cycle:

- clean legacy articles
- standardize metadata/tags
- publish at a higher cadence
- keep technical and reflective writing side by side

## 6. Lessons learned

- Perfection slows output.
- A simple pipeline beats a complex CMS.
- Good archives are built through repeated small edits.
- Writing volume and writing quality can grow together if the system is stable.

## 7. Closing note

This project is evolving from "a personal blog" into a long-term knowledge base.
The next step is disciplined editorial cleanup plus regular publishing rhythm.
